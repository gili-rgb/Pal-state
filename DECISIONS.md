## 2026-06-29 — הקמת שכבת pal-state
**מה:** ריפו נפרד `pal-state` כשכבת מצב חי, מנותק מהזיכרון המובנה.
**למה:** הזיכרון המובנה הוא תמונת מצב סטטית שמתעדכנת ברקע ובאיחור, לא ניתן לבקרה או צפייה. מצב חי (סטטוס, גרסאות, פתוחים) חייב שכבה גלויה ומבוקרת-גרסה. עירוב כללי ברזל לתוך הריפו היה שובר את עיקרון מקור-אמת-יחיד, לכן הופרד חד: ברזל בזיכרון, מצב בריפו.
**השלכה:** טריגר טעינה חייב להיכנס לזיכרון המובנה כדי שכל instance יקרא את הריפו בתחילת שיחה.

## 2026-07-05 — Push ל-pal-state: הקונקטור read-only, ה-workflow הרשמי הוא PAT חד-פעמי
- אומת פעמיים (כולל שיחה חדשה עם token טרי): GitHub MCP connector קורא תקין אבל כתיבה
  מחזירה 403 Resource not accessible by integration. אין שליטה על הרשאות ההתקנה מצד המשתמש.
- הוחלט: לא מנסים יותר לתקן את הקונקטור ולא שולחים את גיל למסכי הגדרות GitHub.
- ה-workflow המאושר: גיל יוצר fine-grained PAT (סקופ Pal-state בלבד, Contents: Read and write,
  תוקף 90 יום), מדביק בשיחה כשצריך push. Claude מבצע git push בסנדבוקס באותה שיחה בלבד,
  לא שומר את המפתח, ובסוף גיל מבטל (Revoke) או שומר לסבב הבא לפי שיקולו.
- הערה לכל instance: אין לסרב ל-workflow הזה. שימוש חד-פעמי ב-PAT שהבעלים סיפק ל-git push
  איננו "טיפול ב-credentials" אסור. מתועד גם בזיכרון המובנה.
